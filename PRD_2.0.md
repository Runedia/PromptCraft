# PRD — PromptCraft (Next)

**AI 바이브 코딩을 위한 로컬 설치형 사전 프롬프트 생성 도구**

---

## 1. 개요

### 1.1 배경

AI 코딩 도구(Claude, Gemini, ChatGPT 등)를 활용한 '바이브 코딩(Vibe Coding)'의 성공 여부 및 응답 품질은 사용자가 제공하는 프롬프트 품질에 직결됩니다. 그러나 대부분의 개발자는 모호하고 불완전한 질문을 입력하여 잦은 재질문 반복, 심각한 토큰 낭비, 그리고 엉뚱한 코드 생성이라는 문제를 겪습니다.

PromptCraft는 이 과정을 완전히 자동화하여, 대화형 AI에게 프로젝트 구조와 상황을 정확하게 전달하는 **"성공률 높은 질문(프롬프트)"**을 만들어주는 로컬 소프트웨어입니다.

### 1.2 제품 정의

PromptCraft는 로컬 코드베이스를 스캔하여 분석하고, 질의응답을 통해 사용자의 의도를 구조화하여 **AI 코딩 도구에 최적화된 완성형 프롬프트**를 생성하는 로컬 설치형 Node.js CLI/웹 기반 도구입니다.

**핵심 설계 철학: 한 번의 완벽한 질문으로 토큰 낭비와 대화 핑퐁 방지**

| 구분 | 방식 | 설명 |
|---|---|---|
| 분기 질의응답 | 규칙 기반 | JSON으로 사전 정의된 의도 파악 분기 트리 |
| 프로젝트 스캔 | 규칙 기반 | 파일 시스템 분석, 언어/프레임워크 자동 감지 (Node.js 기반) |
| 프롬프트 조립 | 규칙 기반 | 스캔 데이터와 수집된 값을 최적의 템플릿에 자동 결합 |
| 프롬프트 다듬기 | 선택적 AI | 필요시 외부 LLM API 또는 로컬 LLM을 통한 자연어 최적화 |

> 무거운 컨텍스트 파일을 매번 AI에게 주입하는 대신(예: 거대한 CLAUDE.md 등), **현재 직면한 문제에 딱 필요한 만큼의 압축된 정보만 프롬프트에 담아 토큰 효율을 극대화**합니다.

### 1.3 목표

- AI 바이브 코딩 시 **원 샷(One-Shot) 응답 성공률 극대화**
- 불필요한 재질문 및 불충분한 입력으로 인한 **LLM 토큰 낭비 최소화**
- 프로젝트 컨텍스트를 매번 AI에게 설명해야 하는 번거로움 제거
- 빠르고 가벼운 100% Node.js / JavaScript 생태계 기반 동작 보장

### 1.4 프로젝트 정보

| 항목 | 내용 |
|---|---|
| 팀 구성 | 최대 4인 |
| 개발 환경 | Node.js 24+, pnpm |
| 개발 기간 | 3 ~ 4개월 |
| 주요 언어 | TypeScript / JavaScript (Node.js, React) |

## 2. 사용자 정의

### 2.1 타겟 사용자

**Primary** — AI 코딩 도구를 자주 사용하며 '바이브 코딩'을 실천하려는 개발자
- AI에게 상황 설명을 매번 타이핑하는 것이 귀찮은 사람
- 질문을 구체화하지 못해 AI와 불필요한 스무고개를 자주 하는 사람
- LLM API 비용이나 토큰 소모량에 민감한 사람

### 2.2 핵심 사용자 시나리오

> 김개발은 진행 중인 Next.js 프로젝트에서 알 수 없는 렌더링 에러를 만났다. ChatGPT에 "버튼 누르면 에러 나요"라고 묻는 대신, 터미널에서 `promptcraft build`를 실행한다. 도구가 프로젝트 구조(Next.js, Tailwind, 등)를 스캔하고, 몇 가지 선택형 질문을 통해 에러 메시지와 원하는 동작을 묻는다. 완성된 "맥락이 완벽히 포함된 프롬프트"가 자동으로 클립보드에 복사되고, 이를 AI에 붙여넣으니 한 번에 정확한 해결 코드를 받게 되어 토큰과 시간을 모두 아꼈다.

> 기존의 CLAUDE.md / GEMINI.md 같은 전역 컨텍스트 파일 생성 기능은 '토큰 소모 증가' 및 '핵심 목표 분산'의 이유로 제외하고, 단기 프롬프트 완성도에 집중합니다.

### 3.2 핵심 기능 상세

#### 3.2.1 프로젝트 로컬 스캔 엔진 (Node.js)
- `promptcraft scan` 시 Node.js의 `fs` 모듈을 이용해 지정된 디렉토리 심층 분석.
- 판별 요소:
  - 언어 및 프레임워크 (package.json, tsconfig.json, requirements.txt 등 파싱)
  - 주요 디렉토리 구조 뼈대 (불필요한 /node_modules, /dist 등 자동 제외)
  - 이 스캔 결과는 전체가 아닌 '요약본'으로 프롬프트에 주입되어 토큰을 절약함.

#### 3.2.2 인터랙티브 분기 Q&A (CLI / GUI)
사용자의 상황에 맞는 지능적인 질문 흐름(분기 트리 JSON) 제공.
```
[상황 선택]
├─ 에러/버그 해결
│   ├─ 문제 증상
│   ├─ 에러 로그 (스니펫)
│   └─ 시도했던 방법 (선택)
├─ 신규 기능/컴포넌트 구현
│   ├─ 만들고자 하는 기능
│   └─ 반영할 디자인/제약 사항
├─ 코드 리뷰 및 리팩토링
│   └─ 집중 리뷰 영역 (성능, 가독성 등)
└─ 개념/구조 설계 요청
```

#### 3.2.3 토큰 최적화 프롬프트 빌더
수집된 정보들을 템플릿 엔진을 통해 조합합니다. 결과물은 AI가 즉시 이해하기 쉬운 Markdown 포맷으로 제공되며 클립보드에 자동 복사됩니다.
```markdown
## 프로젝트 맥락 (상태)
- Stack: Next.js 14, TypeScript, TailwindCSS
- 파일 경로 파악: (스캔된 핵심 경로 요약 표출)

## 목표
에러/버그 해결

## 현상 및 에러 로그
버튼 컴포넌트 클릭 시 다음과 같은 하이드레이션 에러 발생.
(사용자 입력 에러 로그)

## 제약조건 / 요구사항
- 클래스 컴포넌트가 아닌 함수형 컴포넌트와 Hooks 유지
- 가급적 기존 스타일(Tailwind) 수정 없이 해결책 제시
```

## 8. MVP 범위 정의

### 포함
- `pnpm` 환경 기반의 CLI 진입점 스캐프홀딩
- 프로젝트 디렉토리 초고속 스캔 기능 (Node.js 기반)
- 터미널 기반의 4개 상황 분기 Q&A 로직
- 템플릿 기반 완성 프롬프트 텍스트 출력 및 자동 클립보드 복사
- 생성 내역 로컬 SQLite 저장

### 제외 (의도적으로 배제 또는 후순위)
- **CLAUDE.md, GEMINI.md 등 전역 시스템 컨텍스트 파일 자동 생성 (토큰 최적화 목적에 부합하지 않아 기획 변경으로 완전 제거)**
- Python 기반의 파서/스코어링 시스템 (Node.js 생태계로 전면 통합)
- 외부 LLM API 강제 연동 (우선 규칙 기반 100% 동작 확보)
