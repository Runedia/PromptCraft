PRD — PromptCraft 로컬 설치형 코딩 프롬프트 빌더

본 내용은 회의를 통해 변경될 수 있습니다.

1. 개요

1.1 배경

AI 코딩 도구(Claude, Gemini, ChatGPT 등)의 응답 품질은 프롬프트 품질에 직결된다. 그러나 대부분의 개발자는 모호하고 불완전한 질문을 입력하여 재질문 반복, 토큰 낭비, 엉뚱한 답변이라는 문제를 겪는다. 특히 초보 개발자일수록 "무엇을 어떻게 설명해야 하는가"를 모른다.

Claude Code의 SKILL.md처럼, AI 도구에 프로젝트 컨텍스트와 행동 지침을 사전 주입하는 파일이 존재하지만 이를 직접 작성하려면 AI 도구에 대한 깊은 이해가 필요하다. PromptCraft는 이 과정을 자동화한다.

1.2 제품 정의

PromptCraft는 로컬 코드베이스를 분석하고, 질의응답을 통해 사용자의 의도를 구조화하여 완성형 코딩 프롬프트를 생성하는 로컬 설치형 도구다. CLI와 웹 UI를 모두 지원하며, AI 도구별 컨텍스트 파일(CLAUDE.md, GEMINI.md 등)도 자동 생성한다.

핵심 설계 철학: 규칙 기반 우선, AI는 선택적 확장

구분방식설명
분기 질의응답규칙 기반JSON으로 사전 정의된 분기 트리
프롬프트 조립규칙 기반수집된 값을 템플릿에 채우는 방식
파일 스캔 / 언어 감지규칙 기반확장자·설정파일 파싱
컨텍스트 파일 생성규칙 기반스캔 결과 → 템플릿 매핑
프롬프트 자연어 다듬기선택적 AI외부 API 또는 로컬 LLM 연결 시 활성화
품질 스코어링 고도화선택적 AIAPI Key 미입력 시 규칙 기반 스코어링으로 대체

API Key 미입력 시에도 모든 핵심 기능이 완전히 동작한다.

1.3 Claude Skills와의 비교

[Claude Code - SKILL.md]          [PromptCraft]
특정 AI 도구 전용          vs     AI 도구 무관, 범용
사람이 직접 작성           vs     자동 생성
단일 포맷                  vs     도구별 포맷 동시 생성

PromptCraft는 Claude Skills의 로컬 대체제이자, 이를 자동으로 만들어주는 생성 도구다. 동일한 프로젝트 스캔 결과로 여러 AI 도구에 맞는 포맷을 각각 출력한다.

PromptCraft 출력
├─ CLAUDE.md        → Claude / Claude Code용
├─ GEMINI.md        → Gemini용
└─ prompt.txt       → 범용 복사용

1.4 목표

  • 첫 질문에 원하는 답을 얻는 성공률 향상
  • 불필요한 재질문으로 인한 토큰 낭비 최소화
  • 프로젝트 컨텍스트를 매번 수동 입력하는 반복 작업 제거
  • 특정 AI 도구에 종속되지 않는 범용 컨텍스트 파일 생성

2. 사용자 정의

2.1 타겟 사용자

Primary — AI 코딩 도구를 자주 사용하는 개발자 (학생 ~ 주니어)

  • AI에게 질문을 잘 구성하지 못해 비효율적인 대화를 반복하는 사람
  • 매번 프로젝트 구조를 AI에게 다시 설명하는 것이 번거로운 사람

Secondary — AI 도구를 처음 사용하는 비개발 직군

  • 코딩 관련 질문은 아니지만 구조화된 질문 작성이 필요한 사람

2.2 핵심 사용자 시나리오

김철수(대학생)는 Node.js 프로젝트에서 에러를 만났다. 평소처럼 ChatGPT에 "이거 왜 안 돼요?"라고 물으면 엉뚱한 답변이 온다. PromptCraft를 실행하면 프로젝트를 자동 스캔하고, 몇 가지 질문을 통해 에러 메시지, 시도한 방법, 원하는 결과를 수집한다. 완성된 프롬프트를 복사해 AI에 붙여넣으면 정확한 답변이 바로 돌아온다.

3.2 핵심 기능 상세

3.2.1 프로젝트 스캔

  • 지정한 디렉토리를 탐색하여 아래 정보를 자동 수집
    • 주요 언어 (확장자 빈도 기반)
    • 프레임워크 / 라이브러리 (package.json, pom.xml, requirements.txt 등)
    • 디렉토리 구조 (depth 2 수준)
    • .env 존재 여부 (내용 미포함)
  • 스캔 결과는 세션 내 컨텍스트로 유지되어 프롬프트에 자동 삽입

3.2.2 분기 질의응답 엔진

사용자가 선택한 상황 유형에 따라 질문 흐름이 분기된다. 분기 트리는 JSON으로 정의되며 커스텀 추가가 가능하다.

[상황 선택]
├─ 에러 해결
│   ├─ 사용 언어 / 버전
│   ├─ 에러 메시지 (코드 스니펫 직접 입력 가능)
│   ├─ 시도한 방법
│   └─ 원하는 동작
├─ 기능 구현
│   ├─ 구현하려는 기능 설명
│   ├─ 현재 코드 있음 여부
│   ├─ 참고 코드 / 예시
│   └─ 제약 조건 (성능, 라이브러리 제한 등)
├─ 코드 리뷰
│   ├─ 리뷰 포인트 (성능 / 보안 / 가독성 / 전체)
│   └─ 코드 스니펫 입력
└─ 개념 이해
    ├─ 현재 이해 수준
    ├─ 비유 선호 여부
    └─ 관련 코드 있음 여부

3.2.5 선택적 AI 연결 (P2)

핵심 기능은 규칙 기반으로 동작하며, AI 연결은 선택적 확장이다.

[사용자 설정]
├─ API Key 입력 시  → 외부 LLM으로 프롬프트 자연어 다듬기 활성화
├─ Ollama 감지 시   → 로컬 LLM으로 오프라인 AI 기능 활성화
└─ 미설정 시        → 규칙 기반으로만 동작 (모든 핵심 기능 정상 동작)
