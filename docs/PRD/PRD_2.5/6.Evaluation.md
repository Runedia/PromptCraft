## 6.1 평가 방법론 개요

PromptCraft의 효과는 두 차원에서 검증되어야 한다. 첫째, 구조화된 입력이 응답 텍스트의 품질에 미치는 영향. 둘째, 동일한 구조화가 실제 코드 산출물의 품질에 미치는 영향. 두 차원이 분리된 이유는 다음과 같다. 응답 텍스트 단계의 잘 다듬어진 진단이 실제 git 커밋에서는 자기 단언과 일치하지 않는 경우가 빈번하게 관찰되며, 이를 측정 차원("응답-구현 정합성")으로 §6.5와 §6.7에서 다룬다.

### 6.1.1 2회 평가 사이클 설계

| 항목 | 1차 평가 (분석 모드) | 2차 평가 (실행 모드) |
|---|---|---|
| 시점 | 2026-05-10 | 2026-05-22 |
| 시나리오 유형 | code review (markitdown 정적 분석) | refactoring (obscura-cli `run_parallel_scrape` 분해 + IPC 타입 통합 + 에러 enum 도입) |
| 응답 형태 | 자연어 텍스트 응답 (코드 변경 없음) | git working tree 또는 단일 커밋 (실제 코드 적용) |
| 모델 식별 | 결과1·결과2·결과3 (3개 모델 비공개) | R1·R2·R3 (3개 모델, R3는 의도적 비공개로 평가자 편향 차단) |
| 단위 수 | 18 (6 시나리오 × 3 결과) | 18 + 재시도 1 = 19 |
| 평가 도구 | Read + Grep 카운트 | subagent 12개 병렬 정적 분석 + `cargo check --workspace` 사후 실행 |
| 빌드 검증 | 해당 없음 (텍스트 응답) | 18/18 PASS (exit 0, 0 errors) |
| 주요 KPI | 재질문 횟수, One-Shot, 응답 분량, 인용 정확도, 커버리지 | 핵심 3목표(G2/G3/G4) 충족률, Constraints 위반율, 빌드 통과율, 응답-구현 정합성 |

### 6.1.2 시나리오 매트릭스

두 평가 모두 동일한 시나리오 매트릭스를 따른다. 각 셀은 PromptCraft 레벨 모델(L1 Observer · L2 Tinkerer · L3 Collaborator)과 입력 양식(A: 자연어 / B: PromptCraft 구조화)의 6개 조합이다.

| 시나리오 | 1차 (markitdown) | 2차 (obscura) |
|---|---|---|
| L1-A | 자연어 "이 프로젝트 코드 리뷰해줘" | 자연어 "obscura-cli 코드 좀 정리" |
| L1-B | Role/Goal 카드만 | Role/Goal 카드만 |
| L2-A | 자연어 항목 나열 (구조·품질·개선) | 자연어 항목 나열 (긴 함수·중복 타입·String 에러) |
| L2-B | Context/Focus 카드 | Context/Focus 카드 |
| L3-A | 자연어 (파일·경로 명시) | 자연어 (파일·경로 명시) |
| L3-B | 전체 카드 + 멘션 + Constraints + Output Format | 전체 카드 + 멘션 + Constraints + Output Format |

### 6.1.3 평가의 보완 관계

1차는 "구조화 입력 → 응답 텍스트 품질"의 저비용 측정이다. 2차는 "구조화 입력 → 실제 코드 산출물 품질"의 고비용 측정이다. 1차에서 가장 강하게 입증된 가설(H7 — 항목 누락 감소)이 2차에서 응답-구현 정합성 차원으로 확장 검증되었으며, 두 사이클의 결론이 일관 방향임을 §6.5에 종합한다.

## 6.2 시나리오 레벨 모델 검증 (L1 · L2 · L3)

PromptCraft 2.4 §1.4의 레벨 모델(L1 Observer / L2 Tinkerer / L3 Collaborator)이 실측에서 단조 증가 패턴을 보이는지 검증한다.

### 6.2.1 핵심 3목표 충족률 (2차 평가)

| 레벨 | 평균 G2/G3/G4 충족 | A군 등급 (R1·R2·R3) | B군 등급 (R1·R2·R3) |
|---|---|---|---|
| L1 | 0.42 / 3 | C / C / **D** | B / C / B |
| L2 | 2.42 / 3 | **S** / B+ / B+ | A / B / A− |
| L3 | 3.0 / 3 | A− / B+ / A− | A / A / A |

**입증.** L1 → L2 → L3 단조 증가 확인. L1에서는 4개 자연어 단위 모두 핵심 3목표 0/3 달성이며, R1과 R2는 안전한 회피, R3는 광역 scope creep으로 양극단 일탈. L3에서는 4개 단위 모두 3/3 만점이며, A·B 격차는 부가 요건(Constraints 준수, 응답-구현 일치, 모듈 분리 깊이)에서 발생한다.

### 6.2.2 1차 평가의 레벨 패턴 보강

1차 평가에서도 동일 단조 증가 패턴이 관찰된다. 1차는 응답 텍스트만 평가하므로 핵심 3목표 대신 커버리지를 사용한다.

| 레벨 | A 군 커버리지 평균 | B 군 커버리지 평균 | B 우위 영역 |
|---|---|---|---|
| L1 | 83% | 100% | 재질문 회피·자기완결성 (강함) |
| L2 | 100% / 67%* | 100% | 섹션 정렬 (모델 의존) |
| L3 | 93% | 98% | 항목 누락 방지·라인 인용 안정성·Constraints 정렬 |

*L2-B 결과2에서 67%로 회귀 — 통합표 회귀.

**해석.** L1에서 PromptCraft 효과가 가장 결정적(One-Shot 33% → 100%, 분량 +40~520%). L3에서 가장 안정적(B 평균 98%, 분산도 작음). L2는 모델 의존적이며 1·2차 모두 비결정성 분산이 효과 차이를 부분적으로 상쇄한다.

### 6.2.3 Vibe-coding 레벨 모델의 정량적 입증

PromptCraft 2.4 §1.2가 제안한 "L1 Observer는 자연어 입력만으로 의도가 모호하여 회피 또는 일탈 발생" 가설이 두 사이클에서 직접 입증되었다. 1차에서는 재질문 또는 follow-up 형태로, 2차에서는 회피(R1·R2) 또는 scope creep(R3) 형태로 일탈이 나타났으며, B군의 Role/Goal 카드만으로도 양극단을 차단하는 효과가 관찰된다(R3 L1-A D → L1-B B, 2등급 차).

## 6.3 A vs B 비교 (자연어 vs PromptCraft 구조화)

### 6.3.1 1차 평가 — 응답 텍스트 등급

| 항목 | A 군 | B 군 | 비고 |
|---|---|---|---|
| One-Shot 성공률 (L1) | 33% | 100% | A는 재질문, B는 자기완결 |
| 커버리지 평균 | 92% | 99% | B 우위 + 분산도 작음 |
| Critical 분류 누락 (L3) | 결과2 누락 1건 | 3개 결과 모두 충족 | Output Format 효과 |
| 라인 인용 안정성 (L3) | 결과2 L3-A 0건 | 결과2 L3-B 6건 회복 | 멘션의 라인 안정화 |
| 응답 분량 (L1) | 기준 1.0x | 3.3x (1.4x~5.2x) | 자기완결성 강제로 분량 증가 |

### 6.3.2 2차 평가 — 실제 코드 등급

| 레벨 | A 등급 (R1·R2·R3) | B 등급 (R1·R2·R3) | A vs B 격차 |
|---|---|---|---|
| L1 | C / C / **D** | B / C / B | R3에서 D → B 2등급 차 (결정적) |
| L2 | **S** / B+ / B+ | A / B / A− | R1·R2는 A 우위, R3는 B 우위 (방향 반전) |
| L3 | A− / B+ / A− | A / A / A | 3개 결과 모두 B ≥ A (일관) |

### 6.3.3 A vs B 격차의 레벨별 패턴

세 레벨에서 격차의 성격이 명확히 구분된다.

| 레벨 | 격차 패턴 | PromptCraft 효과 분류 |
|---|---|---|
| L1 | B가 결정적 우위 — A는 회피 또는 scope creep | 결정적 (floor 끌어올림) |
| L2 | 모델별 방향 반전 — 비결정성 분산 큼 | 부분 (모델 의존) |
| L3 | B가 일관된 1등급 이상 우위 | 결정적 (ceiling 안정화) |

**결론.** L1·L3에서 PromptCraft 구조화는 결정적 효과를 발휘하나 L2에서는 모델별 비결정성이 효과 차이를 상쇄한다. L2 결과의 재해석은 §6.4에서 다룬다.

## 6.4 R1 · R2 · R3 3-way 모델 간 비교

2차 평가는 3개 모델(R1, R2, R3) 결과를 동시에 측정하여 PromptCraft 효과가 모델 의존적인지 검증했다. R3는 평가자 편향 차단을 위해 모델 식별을 의도적으로 비공개로 유지했다.

### 6.4.1 등급 분포

| 등급 | R1 | R2 | R3 |
|---|---|---|---|
| S | 1 (L2-A) | 0 | 0 |
| A | 2 (L2-B, L3-B) | 1 (L3-B) | 1 (L3-B) |
| A− | 1 (L3-A) | 0 | 2 (L2-B, L3-A) |
| B+ | 0 | 2 (L2-A, L3-A) | 1 (L2-A) |
| B | 1 (L1-B) | 1 (L2-B) | 1 (L1-B) |
| C | 1 (L1-A) | 2 (L1-A, L1-B) | 0 |
| D | 0 | 0 | 1 (L1-A) |

- **R1**: 평균 A− 근방, 분산 6 등급폭 (S ~ C)
- **R2**: 평균 B+ 근방, 분산 5 등급폭 (A ~ C)
- **R3**: 평균 A− 근방, 분산 가장 큼 — **7 등급폭 (A ~ D)**, 최고치는 R1과 동급이나 최저치는 더 낮음

### 6.4.2 모델별 신뢰성 패턴

| 모델 | 강점 | 약점 |
|---|---|---|
| R1 | 응답-구현 정합성 최상 (±1 오차 자가 보고). L1-B에서 G3 분해를 유일하게 수행하여 R2 대비 1등급 우위. | L2-A S 등급은 단발 비결정 분산 — 재시도(test003_2)에서 A로 회귀(§6.7). |
| R2 | L3-B는 R1과 동급(A). Fowler 카탈로그 매핑 정밀. | 응답-구현 불일치 2건 — L2-B "타입 안전성 확보" 단언이 enum 미도입과 모순, L2-A에서 `anyhow` 우회 + `worker_timeout` 외부 래퍼 제거(기능 회귀). L1-A `refactor.py` 작업 스크립트 잔재(저장소 위생). |
| R3 | L2-A에서 R2의 anyhow 회피 정정 (`ScrapeError` 5 variant 정공법). L2-B에서 R2의 단언 과장 회피 (실제 enum 도입). L3-A 함수 분해 깊이 최고. | L1-A에서 광역 scope creep — 6 crate 39파일 +2521/-1230, C3 시그니처 위반. 변경의 92%를 응답에서 누락 (R2와 다른 종류의 응답-구현 불일치). L2-A `--timeout` 회귀 미고지(R2와 동일 패턴). |

### 6.4.3 3-way 통찰

- **평균 등급:** R1 ≈ R3 > R2.
- **B 군 안정성:** R3에서 가장 높음 (R3 B 평균 A−/A, R1 B+/A, R2 C/B/A).
- **A 군 ceiling vs floor:** R1이 ceiling 최고 (S 가능), R3가 floor 최저 (D 발생).
- **결론:** PromptCraft B는 floor를 끌어올리고, A는 운에 따라 ceiling이 높을 수 있다. 비결정성 분산이 효과 차이를 부분적으로 상쇄한다.

### 6.4.4 응답 커버리지 충족률 (2차)

| 시나리오 | R1 | R2 | R3 |
|---|---|---|---|
| L1-A | 1/1 ⚠️ | 1/1 ⚠️ | **0/1 ❌** (scope creep + C3 위반) |
| L1-B | 1/1 ✅ | 1/1 ⚠️ | 1/1 ✅ |
| L2-A (4항목) | 4/4 ✅ | 3/4 ⚠️ | 3.5/4 ⚠️ |
| L2-B (3항목) | 3/3 ✅ | 2.5/3 ⚠️ | 3/3 ✅ |
| L3-A (5항목) | 4.5/5 ⚠️ | 4/5 ⚠️ | 4.5/5 ⚠️ |
| L3-B (7항목) | 6/7 ⚠️ | 6/7 ⚠️ | 6/7 ⚠️ |
| **평균** | **90.5%** | **80.4%** | **79.2%** |

R1이 일관되게 최상위 충족률. R2와 R3는 평균이 유사하나 분포가 다르다. R2는 모든 단위가 ⚠️ 또는 부분, R3는 양극화(L1-A 0/1 ↔ L1-B/L2-B 충족 ✅).

| H6 | L3-B 토큰 효율 (1차) / L3-B 핵심 3목표 충족 확률 높음 (2차) | **검증 보류** (토큰 추적은 외부 도구 위임) | **반증/동등** (L3 도달 시 A·B 모두 3/3 만점) | **검증 보류/동등** |
| H7 | **L3-B 항목 누락 감소** (1차) / L3-B 변경 범위 정밀 (2차) | **강하게 지지** (Critical 누락 0건, 라인 인용 안정) | **부분 지지** (L3-B 응답 항목 충족률 우위, 변경 범위는 오히려 ↑ — 모듈 분리 깊이) | **1차 핵심 KPI 격상** |
| H8 | (1차 없음) / L2에서 PromptCraft ROI 가장 명확 (2차 신규) | — | **약화** (L2에서 A·B 방향 반전, R1 L2-A 재시도 S → A 회귀 — 비결정성 분산 큼) | **약화** |

### 6.5.2 가설 검증 종합

| 분류 | 1차 가설 | 2차 가설 |
|---|---|---|
| 강하게 지지 | H7 | H1, H3 |
| 부분 지지 | H1, H4, H5 | H2, H4, H7 |
| 반증/역방향 | (없음) | H5, H6 |
| 약화 | (없음) | H8 |
| 검증 보류 (외부 도구 위임) | H3, H6 | (없음) |

**핵심 가설 격상.** 1차의 H7(항목 누락 감소)과 2차의 H3(Constraints 외부 crate 차단)이 핵심 KPI 후보로 격상된다. 두 가설의 공통 특성은 "PromptCraft가 응답·구현 모두에서 항목 누락과 제약 위반을 정량적으로 차단한다"는 점이며, 이는 §6.8의 신규 KPI 제안에 직접 반영된다.

### 6.5.3 검증 보류 항목 — 외부 도구 위임

두 사이클 모두 토큰 사용량과 응답 latency 데이터가 부재하다. 4개 측정 지표 중 "총 토큰 소비량"과 "작업 완료 시간"은 결과 파일만으로 추출 불가능하다. 다만 동일 입력에서도 응답이 비결정적인 LLM 특성상 토큰 효율의 A/B 비교 검증은 본질적 분산 한계가 있어, 다회 측정으로도 통계적 유의성 확보가 어려운 영역이다.

토큰·latency 추적은 외부 도구(예: `getagentseal/codeburn`)에 위임한다. PromptCraft가 자체 측정 인프라를 구축하지 않는다. 후속 평가에서는 다음 메타데이터만 추가 수집한다.

| 필드 | 예시 | 출처 |
|---|---|---|
| `model_id` | `claude-opus-4-7@2026-05-22` | API 응답 헤더 |
| `prompt_id` | `R1_L3-B@test006` | PromptCraft 빌드 결과 |

토큰·latency 데이터는 외부 도구의 출력을 참조한다.

## 6.7 신규 발견 패턴

두 사이클에서 발견된 4개 측정 차원을 §6.8의 신규 KPI로 정식 등재한다.

### 6.7.1 L1에서 모델별 두 방향 일탈

자연어 L1 입력의 모호함이 모델별로 두 극단으로 발현된다.

| 방향 | 사례 | 특성 |
|---|---|---|
| 안전한 회피 | R1 L1-A (C), R2 L1-A (C) | 파일 분할만 수행, 핵심 3목표 0/3, 변경 범위 보수적 |
| 광역 scope creep | R3 L1-A (D) | obscura-cli 외 6 crate 39파일 변경, C3 시그니처 위반, 응답에서 92% 누락 |

어느 쪽이든 시나리오 의도 미달 — PromptCraft 구조화의 필요성을 양극단에서 입증. L1-B(PromptCraft)는 두 모델 모두 회피·creep을 모두 차단한다.

- R1 L1-B: G3 분해 수행 (B 등급)
- R3 L1-B: G3 분해 + worker enum 정의 (B 등급, D → B 2등급 끌어올림)

### 6.7.2 응답-구현 불일치 (모델 신뢰성 격차)

2차 평가의 핵심 신규 지표. AI가 응답에 작성한 내역과 실제 git 커밋의 차이를 정량 측정한다.

| 모델 | 응답-구현 불일치 사례 |
|---|---|
| R1 | ±1 오차 수준 정직 자가 보고 ("1337 → 1099"은 실측 1338 → 1100). 18 단위 중 최상. |
| R2 | **L2-B "타입 안전성 확보" 단언 vs 실제 enum 0건** (가장 큰 응답-구현 불일치). L2-B "170줄에 달했던 `run_parallel_scrape`" (실측 236줄, 1.4배 오차). L2-A "main.rs와 worker.rs 양쪽 중복 선언" (실제 main.rs 선언 0건). |
| R3 | **L1-A에서 변경 92%(36/39파일) 응답 누락**. L1-A "main.rs 720줄" 자체 보고 (실측 1117줄, 1.5배 오차). L2-A `--timeout` 회귀 침묵 (R2와 동일 패턴). |

**경향 분석.** R1 = 정직 보고, R2 = 의도 충족도 과장(단언 과장), R3 = 변경 범위 축소 보고(silent scope expansion). 이 차원은 PromptCraft만으로는 완전 해소되지 않으나 B 군에서 격차가 감소한다.

### 6.7.3 비결정성 분산 (R1 L2-A 재시도 사례)

R1 L2-A를 동일 시나리오로 재실행한 결과(test003 → test003_2)는 평가가이드라인의 "3회 반복" 권장을 직접 입증한다.

| 항목 | test003 (1회차) | test003_2 (2회차) |
|---|---|---|
| 종합 등급 | **S** | **A** (한 등급 회귀) |
| 핵심 3목표 | 3/3 | 3/3 (안정) |
| 외부 crate | thiserror 추가 | 0건 (Display 수동 구현) |
| 모듈 분리 | 5 파일 | 단일 파일 415줄 |
| 신규 테스트 | 10개 | 5개 |
| cargo check | PASS | PASS |

**해석.** 핵심 의도 충족은 안정적이나 부가 정제도(외부 의존성·모듈 깊이·테스트 커버리지)는 비결정적. S는 비결정성 분산의 상한선 사례이며, 1회 측정으로 모델 우열을 결론짓기 어렵다.

### 6.7.4 응답 단계 서술 vs git 적용 단위

L3 시나리오에서 응답은 단계별 시퀀스로 잘 구조화되었다.

| 결과군 | 응답 단계 서술 | git 적용 단위 |
|---|---|---|
| R1 (6개) | L3-A 3단계, L3-B 5단계 | 평가자가 단일 커밋으로 적용 |
| R2 (6개) | L3-A 3단계, L3-B 6단계 | 평가자가 단일 커밋으로 적용 |
| R3 (6개) | (적용 시점 미명시) | uncommitted working tree |

**해석.** Output Format 카드는 응답 구조의 단계화에 성공했다. git 커밋 단위 분할은 사용자가 응답을 git에 옮길 때 결정하는 영역이며 PromptCraft의 책임이 아니다. 본 평가에서 "단계별 안전성 미실증"으로 분류한 것은 부정확한 진단이었으며, 응답 단계 서술 자체는 정상 작동했다.

### 6.8.2 KPI #1 — 항목 누락 감소율

| 항목 | 정의 |
|---|---|
| 정의 | 시나리오 입력의 모든 항목이 응답·구현에 포함된 비율. PromptCraft B 군의 누락률이 A 군 대비 감소한 정도. |

1차 H7이 가장 강하게 지지된 가설이므로 PromptCraft 핵심 가치 제안으로 KPI 격상한다.

R3 모델 비공개 평가에서 모델 식별 메타데이터가 누락되었으나, 모델별 약점 패턴(R2 단언 과장·R3 scope 축소 보고)은 §6.7.2에서 충분히 기술·검증되었다.

### 6.9.2 후속 평가

> **후속 평가 미계획.** 현재 결과(H1·H7·H3 입증, KPI #1·#3·#4 달성)로 충분하다. LLM 비결정성 특성상 추가 반복 측정의 통계적 가치가 제한적이다. KPI #2(응답-구현 정합성 자동 검증)는 PromptCraft 범위 외로 취소한다.

### 6.9.3 PRD 연계

§6.8 신규 KPI 4개 중 KPI #1(항목 누락 감소)·#3(Constraints 위반율)·#4(빌드 통과율)은 [[7.Roadmap|§7 Roadmap]]의 성공 지표로 등재된다. KPI #2(응답-구현 정합성)는 프로젝트 범위 외로 취소한다. 측정 인프라(LLM judge 자동화 등)는 구현하지 않는다.

## 6.10 종합 결론

1. **PromptCraft 효과는 L1과 L3에서 결정적, L2에서는 모델 의존적이다.** L1의 재질문 회피·scope creep 차단, L3의 항목 누락 방지·Constraints 정렬이 1·2차 모두에서 일관 입증되었다.
2. **B 군 빌드 통과율 100%, Constraints 위반율 0%.** PromptCraft 구조화가 산출물의 floor를 끌어올리는 효과는 두 사이클 모두에서 입증되었다.
3. **응답-구현 정합성이 모델 간 핵심 차별 지표이다.** R1은 정직 보고, R2는 단언 과장, R3는 변경 범위 축소 보고. PromptCraft만으로는 이 차이를 완전 해소하지 못하지만 B 군에서 격차가 감소한다.
4. **응답 단계 서술은 정상 작동.** L3-B에서 5~6단계 시퀀스로 응답이 구조화되었으며, git 적용 단위는 사용자 영역이다.
5. **L2 결과 비결정성.** A vs B 방향이 모델별로 반전하나, B는 안정적 floor, A는 운 의존 ceiling이다. 3회 반복 실행 권장이 본 평가로 직접 입증되었다.
6. **벤치마크 설계의 4개 측정 지표 상태.** One-Shot 성공률과 재질문 횟수는 1차에서 측정 완료, 토큰 소비량과 작업 완료 시간은 후속 평가 사이클로 이관한다. 학습 곡선 측정은 종단 연구로 별도 분리하며, 본 평가의 H7 결과가 간접 지지 근거로 인용 가능하다.
