## 2026-05-22 — §3.3.2 (단계별 커밋 자동 생성) 삭제

**결정**: "Output Format 카드를 git 다중 커밋으로 확장하는 기능"을 PRD에서 제거.

**근거**: 시나리오 평가에서 "단계별 안전성 18/18 미실증"으로 분석했던 것은 부정확한 진단이었다.

1. 평가 환경에서 git 커밋은 평가자(사용자)가 모델 응답을 직접 적용하면서 수행한 것이다.
2. 모델은 응답을 단계별로 잘 구조화했고(R1 L3-B 5단계, R2 L3-B 6단계), git 적용 단위는 사용자 영역이다.
3. **git 커밋 단위 분할은 PromptCraft의 책임이 아니다.** 모델 응답의 단계 서술까지가 PromptCraft 영역.

**연쇄 정리**:
- §6.7.4 "단계별 안전성 18/18 미실증" → "응답 단계 서술 vs git 적용 단위"로 재정의
- §6.10 결론 4번도 동일 정정
- §7 Roadmap의 P1-1(단계별 커밋 자동 생성), P2-2(단계별 커밋 시나리오 L3-B') 삭제

## 2026-05-22 — §3.3.3 (토큰 사용량 계측 인프라) 삭제

**결정**: 토큰 사용량 자체 측정 인프라(`usage.sqlite` 테이블 등)를 PRD에서 제거.

**근거**:
1. **외부 도구 위임**: `getagentseal/codeburn` 같은 토큰 추적 전용 도구가 이미 존재한다. PromptCraft가 중복 구현할 이유 없음.
2. **본질적 검증 불가능성**: LLM 응답은 비결정적이라 동일 입력·동일 모델에서도 토큰 사용량 분산 큼. R1 L2-A 재시도(`test003_2`)에서 S → A로 등급조차 회귀한 사실이 직접 입증. 토큰 효율의 A vs B 비교는 다회 측정으로도 통계적 유의성 확보 어려움.
3. **인과 추론 불가**: "PromptCraft가 토큰을 절약했다"가 아니라 "응답이 짧았다"만 측정되며, 짧은 응답이 좋은 응답인지도 불명.

**연쇄 정리**:
- KPI #3 (총 토큰 소비량 감소율) 삭제. KPI 18개 → 16개 재번호화.
- §7.2.1 P0 4건 → 1건. 토큰·latency·메타데이터 스펙 3건 삭제, 모델 식별 메타데이터만 P0-1로 유지.
- §7.3 PRD 2.6 예상 영역에서 "토큰 효율 가설 재검증" 삭제.
- H3/H6 "검증 불가" → "검증 보류 (외부 도구 위임)"로 표현 변경.

## 2026-05-22 — 시안 SSOT 폐기, 현재 구현 SSOT 채택

**결정**: `PromptCraftDesigin/` 폴더(시안 HTML/JSX 자료)는 단순 참고였으며, **`src/web/`의 현재 구현이 디자인 시스템의 SSOT**다. 시안 폴더는 PRD 작성 완료 후 삭제 예정.

**근거**:
1. 시안은 현 디자인이 구현된 *이유*를 설명하는 참고자료였을 뿐, 향후 디자인 변경의 기준이 아니다.
2. 시안과 현재 구현의 격차(V1 vs V2 채택, --pc-* vs shadcn 표준 등)는 "결정된 사항"이며 비교 명세가 PRD에 들어갈 가치 없음.
3. PRD §4가 "시안 vs 구현 Delta" 톤이면 시안 폴더 삭제 후 무의미해진다.

**연쇄 정리**:
- §4 UI-Design 통째 재작성. "시안", "PromptCraftDesigin", "V1~V6 변형", "V2 채택", "시안 vs 구현 Delta" 등 비교 표현 모두 제거.
- §3 Features의 "V2 Three-Column 베이스" → "Three-Column" 명칭 단순화.
- §5 Architecture §5.4 토큰 네이밍 표에서 "시안 토큰 vs shadcn" 비교 제거. 현재 구현 명세 + 후속 보강 항목으로 재구성.
- §7 Roadmap의 P1-5~P1-15 디자인 과제에서 시안 출처·권고 인용 제거.

**보존 자료**: 시안에 대한 마지막 분석 기록을 `_drafts/design-{tokens,primitives,screens,interactions}.md` 4개에 보존. 시안 폴더 삭제 후에도 결정 사유 추적 가능.
