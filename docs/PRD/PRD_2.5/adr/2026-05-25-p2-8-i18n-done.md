# 2026-05-25 — §7 P2-8 i18n(ko/en) 구현 완료

**결정**: §7.2.3 P2 표의 P2-8(i18n 지원 ko/en)을 구현 완료로 처리하고 본 표에서 제거한다. P2-8 번호는 번호화 정책에 따라 결번 유지한다. §7.2.4 매트릭스 P2 행 `2건`→`1건`, 총계 `5건`→`4건`. [[0.Index]]:45 §7 요약 과제 건수 `5건`→`4건`.

## 배경 — PRD 정의 대비 범위 확장

PRD §7.2.3의 P2-8 정의는 "카드 레이블·hint·examples 다국어화"로 좁게 기술되어 있었다. 브레인스토밍·구현 과정에서 "출력 프롬프트 헤더 언어 연동"을 사용자가 선택함에 따라, 출력·UI에 흐르는 모든 정의 텍스트가 i18n 대상으로 확정되었다. 실제 구현 범위는 다음으로 확장되었다(설계 문서: `docs/superpowers/specs/2026-05-25-p2-8-i18n-design.md`, 계획: `docs/superpowers/plans/2026-05-25-p2-8-i18n.md`).

| 영역 | 대상 |
|---|---|
| 카드 정의 | label·template·hint·examples·options[].label·options[].value·defaultValue (`data/cards/card-definitions.json` 27개) |
| 도메인 오버레이 | hint·examples (`data/domains/*.json` 7개) |
| role-mappings | domainRoles·languageRoles·frameworkRoles 역할명 (`data/role-mappings.json` 약 227개 항목) |
| trees | label·description·roleSuffix·cardOverrides (`data/trees/*.json` 5개) |
| core 동적 텍스트 | stack-environment 라벨·ts 제약 자연어·role 후보·preview placeholder |
| 웹 chrome | 21개 컴포넌트 약 122개 사용자 대면 문자열 |
| CLI | 8개 출력 문자열 |
| 출력 프롬프트 헤더 | `## Role`↔`## 역할` 등 template 헤더 언어 연동 |

## 핵심 설계 결정

1. **경계 해소(resolution at the boundary)**: 디스크 정의는 인라인 `{ ko, en }`(`I18nText`/`I18nTextArray`)로 양 언어를 보유하고, `createCardSession` 한 지점에서 현재 lang으로 해소(`pickText`/`pickArray`)하여 런타임 `SectionCard`는 단일 string을 유지한다. `buildPrompt` 등 다운스트림은 변경면이 최소다.
2. **무의존 경량 i18n**: 외부 라이브러리 없이 `src/shared/i18n`에 `t(key, lang, vars?)`(점표기 키 + `{{var}}` 보간 + ko→en→key 폴백) + `pickText`/`pickArray` + `detectLocale`. core·server·web·cli 공통.
3. **언어 결정**: config DB `ui.language`(유효 시) 우선, 없으면 OS 로케일(`Intl.DateTimeFormat().resolvedOptions().locale`, `ko*`→ko else en) 감지. server `resolveLang()`이 요청 단위 해소(DB 미초기화 시 OS 폴백). "시스템 따름" = `ui.language` 삭제.
4. **출력 헤더 언어 연동**: template 헤더(`## Role`)를 ko/en 분리해 출력도 UI 언어를 따른다(사용자 결정).
5. **웹 전환**: `LocaleContext`/`useT` + SettingsSheet 언어 3지 선택(시스템/ko/en). 언어 변경 시 `remapResolvedCards`로 정의 필드만 새 언어로 재해소하고 사용자 입력 value·active·order는 보존.

## 신규 구성요소

- `src/shared/i18n/`(types·pickLang·detectLocale·t·locales/{ko,en}.json), `src/web/i18n/`(LocaleContext·useT·resolveInitialLang), `src/server/locale.ts`(resolveLang).
- 신규 라우트: `GET /api/locale`(현재 해소 언어), `DELETE /api/config/:key`(시스템 따름).
- core: `remapResolvedCards`(value 보존 재해소).

## 리스크 (기록)

**평가 baseline 정합성**: [[6.Evaluation|§6]] 시나리오2 평가는 영어 헤더 프롬프트로 수행되었다. lang=ko일 때 출력 헤더가 한국어(`## 역할`)가 되므로, 후속 KPI 재측정 시 baseline(영어 헤더)과의 정합성을 고려해야 한다. 차단 사유 아님. 평가 재현 시 lang=en으로 고정하면 baseline과 동일하다.

## Known limitations (의도된 동작 / 후속 검토)

1. **select 선택 value 미번역**: 사용자가 select 카드(role 등)에서 선택한 옵션의 *value*는 입력값으로 저장되어, 언어 전환 후에도 원래 언어로 유지된다(value 보존 불변식). 드롭다운 라벨은 새 언어로 표시되나 저장값은 불변. 자동 번역하려면 선택 옵션 추적이 필요하여 후속 과제로 둔다.
2. **store.preview 파생값 ko 고정**: `cardStore`의 `derivePromptState`가 `buildPreview(cards)`를 lang 없이 호출해 파생 `preview` 필드의 빈 카드 placeholder가 ko로 고정된다. 단 `PromptPreview`는 cards를 직접 렌더하고 `store.preview`는 undo/redo temporal 스냅샷 전용이라 **화면 영향 없음**.
3. **CLI lang 프로세스 고정**: CLI는 시작 시 `detectLocale()`(메타데이터) / `resolveLang()`(런타임)로 lang을 결정하며, 실행 중 config 변경은 반영되지 않는다(단기 실행 특성상 설계 의도).
4. **`shared/constants.ts`의 QNA_TREE_LABELS/DESCRIPTIONS 한국어**: 현재 외부 참조 없는 dead code로 i18n 누락이 아니다. 향후 재사용 시 i18n 필요.
