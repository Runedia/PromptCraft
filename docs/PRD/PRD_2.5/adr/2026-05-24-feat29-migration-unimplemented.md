# 2026-05-24 — Feature #29 DB 마이그레이션 미구현 확정 및 본문 정정

**결정**: PRD 본문이 "DB 스키마 마이그레이션이 구현됨(현재 스키마 v1)"으로 기술하고 [[7.Roadmap|§7.1.1]] KPI #12를 "✅ 통합 테스트 PASS"로 표기한 것은 코드 실측과 모순이다. 마이그레이션 프레임워크는 **미구현**이다. (1) 본문(§3.2.9·§5.2.3)을 미구현 현실로 정정하고, (2) KPI #12의 ✅ 실측 표기를 철회(미구현 표기)하며, (3) 마이그레이션 구현을 [[7.Roadmap|§7.2.2]] P1 신규 과제 **P1-16**으로 이관한다. [[7.Roadmap|§7.1.2]] 평가 도출 KPI(#13·#15·#16)와 가설(H1~H8)에는 영향이 없다.

**근거**:
1. **코드 실측**: `src/core/db/connection.ts`에 `PRAGMA user_version` 분기·`src/core/db/migrations/` 디렉토리·백업/롤백/기동 중단 로직이 **전부 부재**하며, `CREATE TABLE IF NOT EXISTS history`/`config`만 존재한다(`repositories/`도 `config`·`history` 2종뿐). §3.2.9가 기술한 "`migrations/<version>.ts`의 `up(db)` 함수, 기동 시 자동 적용, 실패 시 `~/.promptcraft/db.sqlite.bak.<ts>` 백업·서버 중단·단일 트랜잭션 DDL 롤백"은 어느 것도 코드에 없다.
2. **선행 발견의 확정**: [[2026-05-24-remove-template-feature|템플릿 제거 ADR]]의 "부수 발견"이 이미 "Feature #29는 PRD가 '구현됨 v1'으로 기술하나 실제 미구현(`connection.ts`에 `user_version`·`migrations/`·백업 로직 부재, `docs/SWA-evaluation-2026-04-30.md:89` 확인)"이라 기록하고 "상태 정정은 별도 결정"으로 보류했다. 본 ADR이 그 별도 결정이다. `template` 테이블 `DROP` 미수행이 마이그레이션 부재의 직접 결과였다는 trail과도 일치한다.
3. **거짓 검증 표기의 위험**: KPI #12 "✅ 통합 테스트 PASS"는 존재하지 않는 기능의 검증을 통과로 단언한 것으로, 평가 신뢰성을 훼손한다. `docs/SWA-evaluation-2026-04-30.md`는 마이그레이션 부재를 위협 S3("기존 사용자 DB 손상 위험", HIGH)로 분류하고 P1 구현(`migrations/v1.ts` + `user_version` + 트랜잭션 래핑 + 백업)을 권고했다. 따라서 KPI 철회와 함께 구현 과제를 P1로 이관한다.
4. **P1 위치 사유**: Feature #29는 [[3.Features|§3.1.2]] 기능 매트릭스에 이미 P1로 선언되어 있으나, 이는 *우선순위 선언*일 뿐 구현 사실이 아니다. §3.2.9 상세 기술이 *구현 완료*처럼 서술되어 모순을 일으켰다. 구현 과제를 §7 향후 과제 P1-A 블록에 P1-16으로 명시하여 "선언된 P1 기능"과 "수행할 P1 과제"를 일치시킨다.

**번호/범위**:
- KPI #12는 취소가 아니라 **미구현 목표 지표로 존치**한다(실측 컬럼만 ✅ → "미구현(P1-16 의존)"으로 정정). 결번 처리하지 않는다.
- P1-16은 P1 시리즈 최대 번호(P1-15) 다음 신규 번호다. 재번호화 없음.
- Feature #29 매트릭스 행(§3.1.2)은 P1로 유지한다(변경 없음).

**연쇄 정리(본문 반영)**:
- [[3.Features|§3.2.9]]: "현재 스키마 **v1**", "`up(db)` 함수, 기동 시 자동, 실패 시 백업·중단·롤백" 기술을 "**미구현**. 현재 `connection.ts`는 `CREATE TABLE IF NOT EXISTS`만 수행하며 버전 관리·백업·롤백이 없다. 마이그레이션 프레임워크 구현은 [[7.Roadmap|§7.2.2]] P1-16 과제"로 정정. 테이블 목록은 실제 `history`·`config` 2종으로 정정(`template`은 [[2026-05-24-remove-template-feature]]로 제거됨).
- [[5.Architecture|§5.2.3]] 성능 표 "DB 마이그레이션 | 기동 시 자동 적용, 실패 시 백업 후 중단" 행: 요구사항을 미구현(P1-16) 목표로 표기.
- [[7.Roadmap|§7.1.1]] KPI #12 행 실측 컬럼 `✅ 통합 테스트 PASS` → `(미구현 — P1-16 과제)`. 해석 문단의 "후반부 10개 지표(#3~#12)는 … 회귀 없이 통과"를 "#3~#11(9개)은 통과, #12(DB 마이그레이션)는 미구현(P1-16)"으로 정정.
- [[7.Roadmap|§7.1.3]] 종합표 자동 검증 행 측정 상태 `✅ 전량 PASS` → `#3~#11 PASS(9개), #12 미구현(P1-16)`.
- [[7.Roadmap|§7.2.2]] P1-A 표에 `P1-16 | DB 스키마 마이그레이션 구현` 행 추가(설명: `PRAGMA user_version` + `migrations/v1.ts` + 단일 트랜잭션 + 실패 시 `~/.promptcraft/db.sqlite.bak.<ts>` 백업·기동 중단. 비고: SWA S3 데이터 손실 위험 HIGH).
- [[7.Roadmap|§7.2.4]] 우선순위 매트릭스 P1-A 행 `2건` → `3건`, 총계 `8건` → `9건`.
