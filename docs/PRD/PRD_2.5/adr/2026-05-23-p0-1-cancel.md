# 2026-05-23 — §7.2.1 P0-1 모델 식별 메타데이터 계획 취소

**결정**: 후속 평가의 P0 과제로 분류돼 있던 `P0-1 모델 식별 메타데이터`(평가 환경 파일에 `model_id`/세션 ID 명시) 도입을 취소. §7.2.1 P0 섹션은 헤딩만 유지하고 "현재 P0 과제 없음" 안내로 대체.

**근거**:
1. 새 평가 사이클에서 `model_id` 명시는 자명한 운영 사항이며, PRD에 P0 과제로 무겁게 둘 사안이 아니다.
2. 모델별 약점 패턴(R2 단언 과장, R3 scope 축소 보고)은 [[6.Evaluation|§6.4]]·§6.7.2에서 이미 기술·검증됨. [[6.Evaluation|§6 본문]]도 line 396에서 "R3 모델 비공개 평가에서 모델 식별 메타데이터가 누락되었으나 모델별 약점 패턴은 충분히 기술·검증되었다"고 명시하여 후속 과제 의존도가 낮음을 본문 자체가 인정.
3. 2026-05-22 [[2026-05-22-token-meter-removed|§3.3.3 토큰 계측 인프라 삭제]] 결정에서 P0 4건이 1건으로 좁아질 때 "토큰·latency는 외부 도구 위임 가능, 모델 식별은 외부 대체 없음"이 단독 잔존 근거였으나, 실측 비공개 평가에서 이미 모델별 패턴 추론에 성공한 점을 들어 본 결정으로 단독 근거가 무효화됨.

**연쇄 정리**:
- [[7.Roadmap|§7.2.1]] P0-1 행 삭제, 섹션 헤딩 유지 + "P0 과제 없음" 명시
- [[7.Roadmap|§7.2.2]] 요약 표 P0 행: `1건` → `0건`, P2 의존성 `P0, P1-1` → `P1-1`
- [[7.Roadmap|§7.2.3]] P2-1·P2-2 의존 컬럼 `P0-1` → `—` (P2-1/P2-2 자체는 유지)
- [[7.Roadmap|§7.3]] 모델별 강점 패턴 카탈로그 의존 `P0-1, P2-2` → `P2-2`
- [[7.Roadmap|§7.5]] 결정 필요 표 토큰 측정 인프라 결정 시점 `P0-1 설계 단계` → `다음 평가 사이클 운영 단계`
- [[6.Evaluation|§6]] 본문은 historical 평가 기록이므로 손대지 않음
- 본 문서 line 130(2026-05-22 §3.3.3 ADR의 "P0 4건 → 1건" 정리)는 historical record로 그대로 유지
