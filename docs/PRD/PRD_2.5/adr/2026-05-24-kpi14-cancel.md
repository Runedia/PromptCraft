# 2026-05-24 — §7 KPI #14 (응답-구현 정합성) · P1-1 자동 검증 과제 취소 정리 (§6 ↔ §7 정합화)

**결정**: §7 Roadmap에 살아있던 KPI #14(응답-구현 정합성) 행과 그 자동화 경로 P1-1(응답-구현 정합성 자동 검증) 과제를 일괄 삭제한다. KPI #14 / P1-1 번호는 번호화 정책에 따라 결번 그대로 유지하며, 본문에는 결번 사유를 1문장씩 명시한다. 정합성을 회복할 기준 결정은 [[6.Evaluation#6.9.2 후속 평가|§6.9.2]] · [[6.Evaluation#6.9.3 PRD 연계|§6.9.3]] 두 절의 기존 취소 결정이다.

**근거**:
1. **§6 ↔ §7 정면 모순**: [[6.Evaluation#6.9.2 후속 평가|§6.9.2]]는 이미 "KPI #2(응답-구현 정합성 자동 검증)는 PromptCraft 범위 외로 취소한다"고 명시했고, [[6.Evaluation#6.9.3 PRD 연계|§6.9.3]]는 "KPI #2(응답-구현 정합성)는 프로젝트 범위 외로 취소한다. 측정 인프라(LLM judge 자동화 등)는 구현하지 않는다"로 결정을 단언했다. 그럼에도 §7.1.2가 동일 KPI를 #14로 등재(목표 "≥ 90% (자동 검증 도입 후 100%)"), §7.2.2 P1-1이 자동화 경로 과제로 잔존(비고 "KPI #14 자동화"), §7.3 PRD 2.6 예상 영역이 "응답-구현 정합성 자동 검증 결과" 1행으로 P1-1 의존을 예고하여 §6의 취소 결정과 정면 모순이었다. 본 결정으로 §7을 §6과 일치시킨다.
2. **취소 방향 채택 사유**: 사용자 확인 — "취소 결정이 맞음". §6.9.2의 기존 근거(LLM judge 자동화는 PromptCraft 범위 외, LLM 응답의 비결정성으로 통계적 가치 제한적, 측정 인프라 미구축 원칙)는 그대로 유효. KPI 등재를 살리고 자동화를 후속 사이클로 미루는 대안은 §6.9.3의 "측정 인프라 구현하지 않는다" 단언과 충돌하여 채택하지 않았다.
3. **번호 결번 유지 정책**: 기존 ADR 패턴 — [[2026-05-23-p2-purge-8|P2 8건 삭제]], [[2026-05-24-p1-14-mention-line-range|P1-14 완료]], [[2026-05-23-theme-and-accent|P1-7·P1-8 완료]]가 모두 번호 결번을 그대로 두고 본문에서만 행을 제거. KPI 번호도 동일 원칙을 적용하여 #13·#15·#16만 잔존, #14는 결번(본문에 1문장 결번 사유 명시). P1-1도 동일.
4. **§7.4 평가 가이드라인 보존 사유**: §7.4는 baseline 정합성 원칙으로 "후속 평가가 수행될 경우"의 가설 번호(H1~H8)·평가 기준(G1~G10)·등급 체계(S/A/B/C/D) 계승을 약속한다. KPI #14 취소는 PromptCraft 측정 인프라 미구축 결정에 한정되며, 평가 baseline 자체에는 영향을 주지 않으므로 §7.4는 손대지 않는다.
