# 2026-05-24 — §7 P1-2 (검증/가드레일 단계 자동 삽입) 과제 취소

**결정**: §7.2.2 P1-A 표에 잔존하던 P1-2(검증/가드레일 단계 자동 삽입 — 빌드 완료 시 "Defensive Thinking" 요구 지시어 선택 삽입) 과제를 삭제한다. P1-2 번호는 번호화 정책에 따라 결번 그대로 유지하며, 본문에는 결번 사유를 1문장 명시한다. 이로써 P1-A "평가 자동화 / 기능 확장" 블록은 응답-구현 정합성 계열 과제(P1-1·P1-2)를 모두 정리하고 P1-3·P1-4만 잔존한다.

**근거**:
1. **유일 근거의 소멸**: P1-2의 정당화는 "시나리오2의 응답-구현 불일치 발견으로 우선순위 상향" 한 문장이었다. 그러나 그 "응답-구현 불일치"는 [[6.Evaluation#6.9.2 후속 평가|§6.9.2]] · [[6.Evaluation#6.9.3 PRD 연계|§6.9.3]]에서 "PromptCraft 범위 외로 취소", "측정 인프라(LLM judge 자동화 등)는 구현하지 않는다"로 결정되었고, [[6.Evaluation|§6 결론]]은 이 현상을 "PromptCraft만으로는 이 차이를 완전 해소하지 못한다"는 모델 신뢰성 격차(R1 정직 / R2 단언 과장 / R3 scope 축소 보고)로 규정했다. 발견 자체가 PromptCraft 범위 외 모델 고유 특성으로 결론난 이상, 그 발견을 근거로 한 P1-2의 우선순위 상향 근거는 소멸했다.
2. **형제 항목 P1-1과의 정합**: 동일 발견에 의존하던 형제 과제 P1-1(응답-구현 정합성 자동 검증)은 [[2026-05-24-kpi14-cancel|KPI #14 · P1-1 취소 ADR]]로 이미 KPI #14와 함께 취소되었다. 불일치를 *측정*하는 P1-1을 범위 외로 잘라내고 같은 발견으로 불일치를 *완화*한다는 P1-2만 P1에 남기는 것은 trail 상 비일관이다. 두 항목을 함께 정리한다.
3. **수단-목적 불일치**: 설령 근거를 인정해도, [[6.Evaluation#6.7.2 응답-구현 불일치|§6.7.2]]가 기록한 불일치의 실체는 "타입 안전성 확보 단언 vs 실제 enum 0건", 줄 수 1.4배 과장, 변경 범위 축소 보고 등 모델의 **자기보고 부정확성**이다. "Defensive Thinking"(에러 처리·엣지케이스·입력 검증을 코드에 반영하라는 지시어)은 코드의 *견고성*을 높이는 수단이며, 모델이 자기가 한 일을 정직하게 보고하게 만드는 수단이 아니다. 진단(자기보고 정직성)과 처방(코드 견고성)이 다른 축에 있어 P1-2는 그 목적을 달성할 수 없다.
4. **근거 교체 후 존치(대안) 미채택 사유**: "응답-구현 불일치" 근거를 "코드 품질(G계열 방어적 패턴) 강화"로 재정의하여 P1-2를 존치하는 대안을 검토했으나 채택하지 않았다. (a) §6이 강하게 입증한 가설은 H7(항목 누락)·H3(Constraints)이며 방어적 코딩 약점을 G계열로 실측한 데이터가 없어 새 근거의 평가적 뒷받침이 부재하다. (b) 제약을 자동 추출·삽입하는 [[7.Roadmap|§7.2.2 P1-3]](자동 제약 조건 매핑)과 가드레일 역할이 중복된다. 따라서 단순 취소가 정합적이다.

**연쇄 정리**:
- [[7.Roadmap|§7.2.2]] P1-A 평가 자동화 표에서 P1-2 행 제거. 기존 P1-1 결번 문장 직후에 "P1-2(검증/가드레일 단계 자동 삽입)는 유일 근거였던 '응답-구현 불일치'가 §6.9.2·§6.9.3에서 PromptCraft 범위 외로 취소되고 §6 결론에서 프롬프트로 해소 불가한 모델 신뢰성 격차로 규정됨에 따라 근거가 소멸하여 취소. P1-N 결번 유지([[DECISIONS]] 참조)" 1문장 추가.
