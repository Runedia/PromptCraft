# 2026-05-23 — §7.2.3 P2 후속 과제 8건 일괄 삭제 (P2-8·P2-10만 잔존)

**결정**: §7.2.3 P2 후속 과제 10건 중 8건(P2-1·P2-2·P2-3·P2-4·P2-5·P2-6·P2-7·P2-9)을 삭제. P2-8(i18n 지원)·P2-10(AI 다듬기) 2건만 잔존. §7.2.3 섹션 제목·intro도 "후속 평가 사이클" → "후속 기능 확장"으로 갱신. P2-N 번호는 번호화 정책에 따라 결번 허용(P2-1~7·9 행 제거, 번호 그대로 둠).

**근거**:
1. **P2-1·P2-2·P2-4 (반복 평가·신규 모델·R3 L1-A 재시도)**: [[6.Evaluation#6.9.2 후속 평가|§6.9.2]]가 이미 "후속 평가 미계획. LLM 비결정성 특성상 추가 반복 측정의 통계적 가치가 제한적"으로 정리했다. §7.2.3에 동일 항목이 잔존하는 것은 §6.9.2와의 정면 모순. 본 결정으로 §7과 §6의 정합성 회복.
2. **P2-3·P2-7 (L1 시나리오 재정의·일탈 자동 분류)**: P2-1/P2-2/P2-4가 삭제되면 후속 평가 자체가 없으므로 분류 도구·시나리오 재정의 항목도 의존 근거 소멸.
3. **P2-5 (release 빌드 검증 확장)**: 시나리오2 평가 한정 단발 후속 작업. [[6.Evaluation#6.9.2 후속 평가|§6.9.2]]의 후속 평가 미계획 결정에 종속.
4. **P2-6 (학습 곡선 — 카드 사용률 감소)**: KPI #2 측정용 종단 연구. 종단 연구는 PRD 범위 외 별도 설계 영역으로 §7.2.3 표에 명시할 운영 항목이 아님. KPI #2 비고는 "종단 연구 별도 설계 필요"로 갱신.
5. **P2-9 (스캔 결과 캐싱)**: 명확한 후속 기능이나 시나리오2 평가에서 성능 측정치(스캔 250 파일 median 11.18ms, threshold 5000ms 대비 2~3 자릿수 여유, [[2026-05-23-nfr-measure-tools|§5.2.3 NFR 측정]] 참조)로 사용자 체감 한계까지 여유 큼. 필요 시점에 재등재 가능.
6. **P2-8·P2-10 잔존 근거**: i18n과 로컬 LLM 후처리는 평가 사이클과 무관한 독립 기능 확장으로 후속 작업 가치가 명확. PRD 2.6 또는 그 이후 검토.
