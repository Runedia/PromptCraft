# 2026-05-25 — §7 P1-15 (empty 카드 editor 측 시각 단서) 과제 취소

**결정**: §7.2.2 C "UI 변형 / 단축키 확장" 표에 잔존하던 P1-15(빈 active 카드용 "EMPTY" mono caption을 `SectionCard` editor 측에 추가) 과제를 삭제한다. P1-15 번호는 번호화 정책에 따라 결번 그대로 유지하며, 본문에 결번 사유를 1문장 명시한다. 이로써 P1-11(완료)·P1-12(완료)에 이어 C 블록의 잔여 과제가 정리되어 C 블록 활성 과제는 0건이 된다.

**근거**:
1. **preview-editor 2열 동시 표시 구조에서의 중복**: `WorkspacePage.tsx`는 `main`(편집 영역)과 `aside w-[440px]`(PromptPreview)을 고정 2열로 항상 동시 렌더하며, preview를 접거나 숨기는 토글이 없다. 빈 active 카드는 `PromptPreview.tsx`에서 이미 `opacity-40` + italic "(비어 있음 — 출력에서 제외)"로 상시 표시된다. 즉 사용자가 카드를 비운 그 순간, 같은 화면 내에서 "출력 제외" 피드백을 이미 받는다. P1-15는 동일 화면 내 중복 단서이며 marginal value가 낮다.
2. **질문 행동 교정의 1차 메커니즘은 이미 editor에 구현됨**: [[1.Overview|§1.2]]가 정의하는 "질문 행동 교정"의 본체는 hint·examples가 "무엇을 채울지"를 시각적으로 강제하는 것이며, 이는 `CardInput`(`SectionCard.tsx`에서 hint·examples를 props로 전달)에 이미 구현되어 있다. P1-15의 "빈 상태면 출력 제외됨"은 부정 피드백 보조 단서일 뿐 핵심 메커니즘이 아니다. P1-15 비고란의 "질문 행동 교정 강화"는 이미 충족된 1차 메커니즘 위의 한계효용 작은 보강이다.
3. **KPI 연결 부재**: KPI #4("빈 헤더 출력 zero")는 *출력* 검증 항목이며 자동 테스트로 이미 PASS다. P1-15는 editor UI cosmetic으로 어떤 KPI에도 매여 있지 않다.

**대안(존치) 미채택 사유**: "사용자가 우측 preview 패널을 보지 않는다"는 케이스를 상정해 P1-15를 존치하는 안을 검토했으나 채택하지 않았다. (a) preview는 토글 없는 상시 패널이므로 해당 케이스 자체가 현 레이아웃에서 성립하지 않는다. (b) 설령 향후 preview를 접을 수 있게 한다면, 그것은 P1-15(editor 단서 중복 추가)가 아니라 레이아웃·패널 가시성 설계로 다룰 문제다.

**연쇄 정리**:
- [[7.Roadmap|§7.2.2 C]] 표에서 P1-15 행 제거. 표에 잔여 행이 없으므로 B 블록과 동일하게 산문으로 전환하고, 도입 문장 + "현재 해당 분류의 미해결 과제는 없습니다" + P1-15 결번 사유 1문장(취소 이력은 본 ADR 참조)을 기재한다. 기존 P1-11·P1-12 결번 trail 문장은 유지한다.
- [[7.Roadmap|§7.2.4]] 우선순위 매트릭스: P1-C 행 `1건`/`독립` → `0건`/`—`, 총계 `6건` → `5건`.
- [[0.Index]]:45 §7 요약 과제 건수 `8건` → `5건` 갱신. 이 값은 [[2026-05-25-p1-12-settings-ui-done|P1-12 ADR]](총계 8건→6건) 시점에 갱신 누락된 stale이었으며, 본 결정의 5건으로 바로잡으면서 누락분을 함께 흡수한다. 같은 행의 "15개 KPI"는 현 상태와 일치하므로 유지하고, [[0.Index]]:41 §3 요약의 "31개 P0/P1/P2"는 [[2026-05-24-section-7-3-purge]]가 이미 별도 보고한 선행 stale로 본 결정 범위 외다.
- 본 ADR로 인한 dead link 검토: PRD 본문 내 P1-15 직접 cross-ref는 §7.2.2 C 표 행 1건뿐이며 본 정리로 제거된다. ADR 내 P1-15 참조([[2026-05-22-design-source-of-truth]], [[2026-05-24-p1-11-history-done]], [[2026-05-24-feat29-migration-unimplemented]])는 모두 작성 시점 상태를 기록한 historical record이며, 결번 정책에 따라 P1-15 번호가 유지되므로 dead link가 되지 않는다. 그대로 보존한다.
