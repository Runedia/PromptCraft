# 2026-05-22 — §3.3.2 (단계별 커밋 자동 생성) 삭제

**결정**: "Output Format 카드를 git 다중 커밋으로 확장하는 기능"을 PRD에서 제거.

**근거**: 시나리오 평가에서 "단계별 안전성 18/18 미실증"으로 분석했던 것은 부정확한 진단이었다.

1. 평가 환경에서 git 커밋은 평가자(사용자)가 모델 응답을 직접 적용하면서 수행한 것이다.
2. 모델은 응답을 단계별로 잘 구조화했고(R1 L3-B 5단계, R2 L3-B 6단계), git 적용 단위는 사용자 영역이다.
3. **git 커밋 단위 분할은 PromptCraft의 책임이 아니다.** 모델 응답의 단계 서술까지가 PromptCraft 영역.

**연쇄 정리**:
- §6.7.4 "단계별 안전성 18/18 미실증" → "응답 단계 서술 vs git 적용 단위"로 재정의
- §6.10 결론 4번도 동일 정정
- §7 Roadmap의 P1-1(단계별 커밋 자동 생성), P2-2(단계별 커밋 시나리오 L3-B') 삭제
