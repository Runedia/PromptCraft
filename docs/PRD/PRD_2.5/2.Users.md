# §2 Users

PromptCraft의 대상 사용자, 레벨별 사용 패턴, 그리고 도구 사용에 따른 행동 변화 가설을 정의합니다. 시나리오2 평가(R1·R2·R3 × L1·L2·L3 × A·B)에서 입증된 정량 데이터를 반영하여 레벨별 효과 차이와 행동 전환 경로를 명시합니다. 정량 출처는 [§6 Evaluation](./6.Evaluation.md)을 참조하십시오.

---

## 2.1 사용자 정의 / 페르소나

### Primary 페르소나

AI 코딩 도구를 일상적으로 사용하며 "바이브 코딩"을 실천하려는 **L2~L4 개발자**.

다섯 가지 페인 포인트를 공유합니다.

- AI에게 상황 설명을 매번 타이핑하는 것이 귀찮은 사람
- 질문을 구체화하지 못해 AI와 불필요한 스무고개를 자주 하는 사람
- LLM API 비용 또는 토큰 소모량에 민감한 사람
- 프롬프트 보조 도구가 또 다른 API 비용을 발생시키는 구조에 불만을 가진 사람
- `CLAUDE.md` 등 규칙 파일을 작성해도 막상 질문 시 의식하지 못해 여전히 모호하게 묻는 사람

### 김개발 시나리오 — 핵심 사용 흐름

김개발은 Next.js 14 + Bun 기반 프로젝트에서 헤더 컴포넌트의 빌드 에러를 디버깅하려 합니다. 터미널에서 `promptcraft serve`를 실행하면 브라우저가 자동으로 열립니다. 다섯 개의 트리 중 "에러 해결"을 선택하고, 프로젝트 경로를 입력합니다. 입력 후 800ms debounce를 거쳐 Pre-scan이 트리거되며, 도메인이 `web-frontend`로 분류되면서 `stack-environment` 카드가 자동으로 채워집니다(Next.js 14, Bun). `role` 카드에는 도메인과 트리 기반의 역할 후보 칩이 노출됩니다(예: "프론트엔드 디버깅 전문가").

김개발은 `error-evidence` 카드에 `@src/components/Header.tsx:40-55`를 입력합니다. 서버 사이드 자동완성이 파일 경로와 라인 범위를 즉시 보조합니다. 실시간 프리뷰가 갱신되고, 카드 풀에서 "빌드 로그" 카드를 드래그하여 배치합니다. 마지막으로 **[Run as Claude Code]** 버튼을 누르면, 서버가 cwd를 설정하고 `.promptcraft/runs/<runId>.prompt.md`에 프롬프트를 저장하며 `claude` CLI를 detach spawn 합니다. 실행 로그는 `.promptcraft/runs/<runId>.log`에 기록됩니다. 브라우저 전환·복사·붙여넣기 단계는 완전히 제거됩니다.

### 페르소나 범위에 대한 보조 노트

시나리오2 평가에서 L1 사용자(큰 덩어리 요청, 카드 미숙)는 PromptCraft 미적용 시 두 방향의 일탈을 보였습니다 — 회피적 재질문(R3 L1-A D 등급) 또는 무차별적 scope creep. 이는 L1 사용자가 PromptCraft의 직접 타겟은 아니나, 도구가 제공하는 "구조화 강제"를 통해 부분적 보호 효과를 받는 부수 수혜자(secondary beneficiary)임을 시사합니다. 자세한 정량 근거는 §6.3을 참조하십시오.

## 2.2 사용 시나리오 매트릭스

시나리오2 평가는 사용자 입력 수준(L1·L2·L3)별로 PromptCraft 적용(A) 대비 미적용(B)의 효과 격차를 측정했습니다. 결과는 레벨별로 비대칭적이며, 본 절은 그 격차를 토대로 권장 사용 시나리오를 매트릭스화합니다.

### L1 — 큰 덩어리 요청 / 카드 미숙

**입력 특성**: "이거 안 돼요", "오류 좀 봐주세요" 같이 맥락·증거·범위가 모두 결여된 요청. 카드 구조에 대한 인식이 거의 없음.

**PromptCraft 미적용(B) 시 위험**: R3 L1-A 미적용 케이스에서 D 등급이 관측되었습니다(§6 Evaluation 참조). 모델이 가용한 빈약한 단서로부터 두 방향 중 하나로 일탈합니다 — (a) 회피적 재질문 반복으로 진척 없음, 또는 (b) 임의 추론으로 scope를 부풀려 의도와 무관한 영역까지 수정 시도(scope creep).

**PromptCraft 적용(A) 효과**: 결정적이지는 않으나 **"재질문 회피"와 "scope creep 방지"가 핵심 가치**입니다. 카드 구조가 최소한의 맥락(파일/도메인/스택)을 강제 입력하게 함으로써, 모델이 작업 경계를 임의 추정하지 않도록 보호합니다. 효과의 본질은 "응답 품질 상승"보다 "치명적 실패 회피"에 가깝습니다.

### L2 — 항목 나열 / 추상 표현

**입력 특성**: "헤더 컴포넌트 스타일이 깨졌어요. tailwind 클래스 충돌인 것 같은데, 반응형도 같이 봐주세요" 등 항목은 나열하나 파일·라인·@멘션 같은 결정적 단서가 부재.

**PromptCraft 효과**: **비결정적**. 시나리오2에서 A·B 격차는 좁았고, 풍부한 자연어만으로도 모델이 일부 항목을 충족하는 경우가 빈번했습니다. PromptCraft는 일관성과 누락 방지에는 기여하나, "사용 안 했을 때도 어느 정도 되는" 영역입니다. 따라서 L2 사용자에게 PromptCraft는 **품질 안정화 도구**의 성격이 강하며, 도구의 ROI는 사용자가 얼마나 항목 누락 비용에 민감한지에 따라 달라집니다.

### L3 — 파일·라인·@멘션 명시

**입력 특성**: `@src/components/Header.tsx:40-55`, 명시적 제약(이 함수만 수정, 테스트 보존, 의존성 추가 금지), 명확한 성공 조건.

**PromptCraft 효과**: **결정적**. 시나리오2에서 L3-A는 Constraints 위반 0%, 항목 누락 유의 감소가 관측되었습니다. 카드 구조가 L3 사용자의 정제된 의도를 손실 없이 모델로 전달하는 채널 역할을 하며, 사용자의 작성 비용도 자유 텍스트보다 낮아집니다. **L3 사용자가 PromptCraft의 가치를 가장 명확히 체감하는 그룹**이며, 도구의 권장 대상 핵심입니다.

### 매트릭스 요약

| 레벨 | 입력 특성 | PromptCraft 효과 성격 | 주된 가치 |
|---|---|---|---|
| L1 | 큰 덩어리 / 카드 미숙 | 보호적·부분적 | 재질문 회피, scope creep 방지 |
| L2 | 항목 나열 / 추상 | 비결정적 / 안정화 | 누락 방지, 일관성 |
| L3 | 파일·라인·@멘션 | 결정적 | Constraints 0% 위반, 손실 없는 의도 전달 |

## 2.3 행동 변화 가설

PromptCraft는 단순히 "더 좋은 프롬프트를 한 번 생성하는 도구"가 아니라, **반복 사용을 통해 사용자의 질문 행동 자체를 점진적으로 구조화된 형태로 전환시키는 학습 매체**라는 가설을 둡니다.

### H-Users-1: 카드 구조의 내재화

**가설**: PromptCraft를 반복 사용한 사용자는 카드 단위(role·goal·constraints·evidence)로 작업을 분해하는 사고를 내재화하며, 도구 외부 환경(예: 다른 IDE의 chat, 동료에게 질문할 때)에서도 유사한 구조로 요청을 생성하기 시작합니다.

**지지 근거(부분)**: 시나리오2 평가는 일회성 사용에 대한 A/B 비교이므로 종단적(longitudinal) 행동 변화 자체는 직접 입증하지 못합니다. 다만 **L3 입력의 정량적 우위(Constraints 0% 위반)가 명확히 입증**되었으므로, "L1·L2 사용자가 L3 수준의 입력을 작성하도록 유도되면 효과 격차가 자동으로 닫힌다"는 메커니즘은 데이터로 뒷받침됩니다. 즉, 행동 전환의 **목적지가 가치 있다는 것**은 입증되었고, **전환 경로 자체**가 종단 연구로 검증 대상입니다.

### H-Users-2: L1 → L2 → L3 점진 전환

**가설**: PromptCraft 사용 빈도가 누적될수록 사용자는 다음 경로로 이동합니다.

1. **L1 → L2**: 카드의 존재를 인지하고, 항목 단위로 요청을 분리하기 시작.
2. **L2 → L3**: `@mention` 자동완성과 evidence 카드의 반복 사용을 통해 파일·라인 명시가 습관화. 카드 풀에서 직접 카드를 선택·재배치하며 자기 작업 패턴을 발견.

**지지 근거(부분)**: 시나리오2의 레벨별 효과 격차는 "L3 입력이 우월하다"는 사실을 입증합니다. 사용자가 L3로 이동할 동기를 도구가 어떻게 제공할지는 §5 Roadmap의 온보딩·튜토리얼 항목과 연결됩니다.

### H-Users-3: 부수 수혜자로서의 L1

**가설**: L1 사용자는 PromptCraft의 직접 타겟이 아니더라도, 카드 구조의 최소 입력 강제만으로 **모델의 치명적 실패(scope creep, 무한 재질문)를 회피**합니다. 이는 "품질 상승"이 아닌 "하방 위험 차단"의 효과이며, 도구의 가치 정의를 확장합니다.

**지지 근거**: R3 L1-A D 등급의 정량 관측(§6.3). PromptCraft 미적용 시 L1 입력에서 두 방향 일탈이 재현됨.

### 가설 검증의 한계

본 절의 가설들은 시나리오2의 횡단(cross-sectional) A/B 비교만으로는 완전 검증되지 않습니다. 종단 연구(longitudinal cohort study)와 사용 빈도-입력 품질 상관 분석이 향후 평가 라운드에서 필요합니다. 그러나 **L3 입력의 결정적 우위**가 정량 입증된 이상, 행동 전환의 종착점 가치는 명확합니다. 도구 설계는 그 종착점으로의 유도(affordance)를 우선순위로 삼습니다(§5 Roadmap 참조).
