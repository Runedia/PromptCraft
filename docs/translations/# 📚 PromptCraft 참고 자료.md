# 📚 PromptCraft 참고 자료 및 설계 전략

## 1. 참고 자료 조사 목적
생성형 AI는 사용자의 질문 방식에 따라 결과 품질 차이가 크며, 특히 코딩 초보자나 비전공자는 원하는 답을 얻기 위해 질문을 여러 번 수정해야 하는 경우가 많습니다. 본 프로젝트는 이러한 문제를 해결하기 위해 사용자의 질문을 구조화하여 AI가 의도를 정확하게 이해하도록 돕는 프롬프트 시스템 설계를 목표로 합니다. 이를 위해 최신 AI 에이전트 설계 방식과 프롬프트 구조화 사례를 조사하였습니다.

---

## 2. 참고한 기술 흐름 및 자료

### ① Harness Engineering (하네스 엔지니어링)
최근 AI 시스템은 단순 프롬프트 작성보다, AI가 안정적으로 동작하도록 만드는 시스템 구조(Harness)에 집중하는 방향으로 발전하고 있습니다.
* **주요 내용:** 프롬프트 자체보다 실행 구조 설계 중요성(Planning → Verification), Self-Correction 기반 응답 안정성 확보.
* **적용 목적:** AI가 단순 응답 생성이 아니라, 스스로 답변을 검증하도록 설계.

### ② MCP (Model Context Protocol)
AI가 외부 도구(DB, 검색, 문서)를 사용할 때 신뢰도 높은 정보를 가져오기 위한 표준 인터페이스 구조를 조사하였습니다.
* **주요 내용:** AI와 외부 데이터 연결 방식 표준화, Tool 기반 검증 구조를 통한 환각(Hallucination) 감소.
* **적용 목적:** 모르는 내용을 추측하지 않고 검색 또는 자료 기반 응답 유도.

### ③ Agentic Workflow (에이전틱 워크플로우)
AI가 단순 질문-답변 모델이 아니라, 문제를 단계적으로 해결하는 흐름을 가지도록 설계하는 방식입니다.
* **주요 내용:** 문제 분석, 해결 계획 수립, 도구 선택, 결과 검증의 단계적 프로세스.
* **적용 목적:** 질문이 애매해도 의도를 추론하여 정확한 답변 생성.
