규칙 파일은 "AI의 프로젝트 이해"만 자동화. "사용자의 L2→L3 전환"은 빈 카드 시각 압박만이 강제합니다.

PART 3 · 입증 — 평가 설계

2회 평가 사이클 · 18단위 매트릭스

1차 — 분석 모드

  • code review (markitdown)
  • 자연어 텍스트 응답 품질
  • 재질문·커버리지·인용 정확도

2차 — 실행 모드

  • refactoring (obscura-cli)
  • 실제 코드 적용 + cargo check
  • 핵심 3목표·제약 위반·빌드 통과
매트릭스 : 3 모델 (R1·R2·R3) × 3 레벨 (L1·L2·L3) × 2 입력 (A 자연어 / B 구조화) = 18 단위. R3는 평가자 편향 차단 위해 모델 비공개.
PART 3 · A vs B 비교

자연어 vs 구조화 — 레벨별 등급

레벨A 자연어 (R1·R2·R3)B 구조화 (R1·R2·R3)효과
L1 낮은 숙련C / C / DB / C / B결정적 (R3 D→B)
L2 중간S / B+ / B+A / B / A−부분 (방향 반전)
L3 높은 숙련A− / B+ / A−A / A / A결정적 (전부 B≥A)
정직한 한계 : L1·L3은 결정적이나 L2는 모델 비결정성이 효과를 상쇄(방향 반전). L1-A는 "안전한 회피"(R1·R2 C) 또는 "광역 scope creep"(R3 D, 39파일·변경 92% 누락) 양극단으로 일탈.
