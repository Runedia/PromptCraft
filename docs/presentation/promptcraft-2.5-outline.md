> 본 문서는 발표 **목차 + 작성 가드레일**이다. 슬라이드 상세 대본은 별도 작성한다.

## 경쟁 비교 (슬라이드 4 상세)

PromptCraft는 서로 다른 **세 분류**를 동시에 능가한다.

| 비교 대상 | 그들의 한계 | PromptCraft의 우위 |
|----------|-----------|-------------------|
| ① 프롬프트 생성·최적화 도구 | 생성·보정에 LLM API 사용 → **이중 비용**(생성 1회 + 실행 1회, 실패 시 추가) | 생성 단계 **LLM-free** → 실제 비용 발생 단 1회 |
| ② AI 도구 규칙 파일 (`CLAUDE.md`, `.cursorrules`, `copilot-instructions.md`) | **프로젝트 자동 컨텍스트**만 해결. 사용자 **질문 행동은 미교정** | 빈 카드 시각 압박으로 **질문 행동 L2→L3 교정** |
| ③ LLM 직접 입력 (웹 채팅) | 구조 없음, 코드 외부 전송 | **100% 로컬**(localhost 격리) + Run as 직접 실행으로 복사-붙여넣기 제거 |

→ §1.2 **이중 가치 제안**: ① 비용 제로(기술적 가치) + ② 질문 행동 교정(행동적 가치).

## ⚠️ 모순 방지 가드레일 (슬라이드 작성 시 필수 준수)

1. **런타임은 Bun으로 통일**
   - 2.5: **Bun 1.3.10+** / `bun:sqlite` (Node.js 단일 의존 폐지). 서버 프레임워크는 **Express 5** 유지.
   - 2.3 자료의 "Node.js 24+ / pnpm / better-sqlite3" 표현은 **폐기**. 발표에서 사용 금지.

2. **토큰 절감 정량 수치 사용 금지**
   - 2.5 §1.6: 토큰 사용량 추적은 **범위 외**(외부 도구 위임), 자체 계측 인프라 미구축.
   - 토큰 효율 A/B 비교는 "LLM 비결정성에 따른 **본질적 분산 한계**"로 결정적 입증 영역에서 **제외**.
   - "70% 시간 단축", "토큰 X% 절감" 등 **미입증 수치 금지**. → "후속 평가 사이클에서 정량 검증 예정"으로 처리.
   - 입증된 것은 **결정적 영역**뿐: L1 scope creep 차단, L3 Constraints 위반율 0%, 항목 누락 감소.

3. **AI 다듬기 경계 명확화**
   - "**코어 생성 플로우는 LLM-free**, AI 다듬기(P2-32)는 사용자가 명시적으로 켜는 **선택적·로컬·옵트인 후처리**"로 프레이밍.
   - (주의) AI 다듬기 기능은 현재 코드 수정 진행 중 — 슬라이드 작성 시 최신 구현 상태 재확인 필요.

## 검증 데이터 팩트시트 (인용 가능한 정확 수치)

> 출처: §1.4.1, §3.1, §3.2.1, §6. 슬라이드의 모든 정량 주장은 아래 범위 내에서만 인용한다.

**평가 대상·설계**
- 과제: obscura-cli `run_parallel_scrape` 분해 (Rust workspace, refactoring 트리)
- 일자/라운드: 2026-05-22 시나리오2(실행 모드)
- 매트릭스: R1·R2·R3 3개 모델 × L1-A/L1-B/L2-A/L2-B/L3-A/L3-B 6시나리오 = **18단위**
- 1차 평가: code review 3회 반복 / 2차 평가: refactoring 18단위

**핵심 3목표(G2/G3/G4) 충족률 — 평균**
| 레벨 | A군(자연어) | B군(구조화) | B−A |
|---|---|---|---|
| L1 | 0.0 / 3 (3모델 모두) | 1.0~1.5 / 3 | +1.0 ~ +1.5 |
| L2 | 2.42 / 3 | 2.5 / 3 | +0.0 ~ +0.5 (모델별 반전) |
| L3 | 3.0 / 3 | 3.0 / 3 | 동률(등급차는 부가 요건에서 발생) |

**L1 두 방향 일탈(자연어 L1-A의 모호함)**
- 안전한 회피: R1 L1-A (C등급), R2 L1-A (C등급) — 핵심 3목표 0/3, 보수적
- 광역 scope creep: R3 L1-A (D등급) — 6개 crate 39파일 +2521/-1230, C3 시그니처 위반, 변경의 92% 응답 누락
- L1-B(구조화)는 회피·creep 모두 차단. **R3에서 D → B 2등급 상승**.

**L3 등급(부가 요건)**
| 모델 | L3-A | L3-B | 격차 |
|---|---|---|---|
| R1 | A− | A | +1 |
| R2 | B+ | A | +1 |
| R3 | A− | A | +1 |

**결정적 지표**
- Constraints 위반율: **B군 0% (0/18)** vs **A군 11.1% (2/18)**
- 빌드: 18단위 모두 `cargo check --workspace` exit 0 / 컴파일 에러 0건
- H7(출력 항목 누락 감소): 두 평가 사이클 모두에서 **가장 강하게 입증**

**구현 상태(§3.1, ✅ = 평가로 검증)**
- 27카드 / 5트리(`error-solving`·`feature-impl`·`code-review`·`concept-learn`·`refactoring`) / 11도메인 6키 매핑 / 3계층 오버레이
- @file 멘션 라인범위 `@path:start-end` ✅
- Run as 4 Provider(Claude Code·Gemini·Copilot·Codex), fire-and-forget
- Three-Column(좌 240 / 중 flex / 우 440px)
- 보안: CSP / CORS / 경로 순회 차단 / `.env` 차단
- 라이트/다크 토글(shadcn baseColor: zinc), i18n(ko/en)
